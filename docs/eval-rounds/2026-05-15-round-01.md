## Per-session preflight (your shell, *before* `claude`)

Export the env vars the harness hooks read:

```bash
export SEER_EVAL_RUN_ID=2026-05-15-round-01
export SEER_EVAL_ARM=A     # or C for the second session of a set
```

For an **arm-A session only**, also disable the `repo-map` skill so the
operator-Claude cannot even see it (defense-in-depth on top of the
prompt-level forbid):

```bash

mv .claude/skills/repo-map .claude/skills/_repo-map.disabled
```

Then launch `claude` from the seer-cli repo root. After the arm-A
session, restore the skill before the arm-C session:

```bash
mv .claude/skills/_repo-map.disabled .claude/skills/repo-map
```
